6.3.1.2. Puerto lógico

En la capa de transporte del modelo OSI, el protocolo TCP (Transmission Control Protocol) es uno de los principales responsables de garantizar una comunicación confiable entre dos dispositivos en una red. La multiplexación permite que varias aplicaciones usen simultáneamente el mismo canal de comunicación (una red). La capa de transporte usa números de puertos para identificar qué aplicación debe recibir los datos. Un concepto clave en TCP es el puerto lógico, que se utiliza para identificar de manera específica a qué aplicación o servicio en un dispositivo se debe entregar una cierta comunicación.

Un puerto es un número asociado a una aplicación o servicio específico dentro de un dispositivo. Por ejemplo, cuando enviamos un correo electrónico o accedemos a una página web, los datos no solo necesitan llegar a la IP correcta, sino que también deben ser dirigidos al programa o servicio adecuado dentro de esa máquina. Los puertos hacen esta función.

Cuando un paquete llega a un dispositivo, la demultiplexación en la capa de transporte se encarga de dirigir los datos al proceso o aplicación correspondiente, basándose en los números de puerto.

Por ejemplo, cuando enviamos un correo electrónico o accedemos a una página web, los datos no solo necesitan llegar a la IP correcta, sino que también deben ser dirigidos al programa o servicio adecuado dentro de esa máquina. Los puertos hacen esta función.

  • Puerto 80: Es el puerto estándar para el protocolo HTTP (navegación web).
  • Puerto 443: Es el puerto utilizado para HTTPS (navegación web segura).
  • Puerto 25: Usado para SMTP (correo electrónico).
  • Puerto 53: Usado para DNS (sistema de nombres de dominio).

¿Qué es un puerto lógico?

Un puerto lógico es un número asociado a una aplicación o servicio en un dispositivo dentro de una red. Los números de puerto permiten que múltiples aplicaciones en un mismo equipo puedan usar simultáneamente el protocolo TCP (o UDP, que también usa puertos) sin interferirse entre sí. Un puerto es básicamente un punto de conexión en un dispositivo para diferenciar las diversas conexiones.

¿Cómo funciona el puerto en TCP?
Cuando un dispositivo se conecta a otro a través de TCP, la conexión se define por cuatro elementos:

  1. Dirección IP del origen (el dispositivo que inicia la conexión).
  2. Puerto de origen (el puerto desde donde se origina la conexión en el dispositivo cliente).
  3. Dirección IP del destino (el dispositivo que recibe la conexión).
  4. Puerto de destino (el puerto en el dispositivo receptor que espera las conexiones TCP).

Por ejemplo, si un usuario en su navegador intenta conectarse a un sitio web, la solicitud TCP tiene la siguiente estructura:

  • Dirección IP de origen: La IP del cliente (tu computadora).
  • Puerto de origen: Un número de puerto efímero generado por el sistema operativo del cliente (por ejemplo, el puerto 49152).
  • Dirección IP de destino: La IP del servidor web.
  • Puerto de destino: El puerto 80 (HTTP) o 443 (HTTPS), que son los puertos estándar para estos servicios.

Tipos de puertos

  • Puertos bien conocidos (0-1023): Son números de puerto asignados a servicios estándar, como el puerto 80 para HTTP o el 443 para HTTPS.
  • Puertos registrados (1024-49151): Estos puertos pueden ser utilizados por aplicaciones específicas, pero no son tan universalmente reconocidos como los puertos bien conocidos.
  • Puertos efímeros (49152-65535): Son utilizados de manera temporal por el sistema operativo para asignar conexiones salientes, como las que realiza un cliente para establecer una comunicación con un servidor. Por ejemplo, supongamos que un cliente desea conectarse a un servidor web (www.example.com) usando el protocolo HTTPS. El cliente utilizará un puerto efímero en su máquina, como el 50321, para enviar la solicitud al puerto 443 del servidor, que es el puerto estándar para conexiones seguras HTTPS. Esto significa que el cliente está pidiendo al servidor que entregue los datos a la aplicación que está "escuchando" en el puerto 443. Al mismo tiempo, el servidor sabrá enviar la respuesta a la IP y puerto de origen del cliente (en este caso, 50321).

Ejemplo detallado de TCP en acción

Imaginemos que estás accediendo a una página web en tu navegador:

  1. Tu navegador (cliente) solicita la página web a través del protocolo HTTP.
  2. Se utiliza TCP para establecer una conexión con el servidor web usando el Three-Way Handshake.
  3. El servidor envía los datos (la página web) en pequeños segmentos.
  4. Tu navegador recibe los segmentos y TCP se asegura de que lleguen en el orden correcto, reensamblando la página.
  5. Cuando todos los datos han sido transferidos, TCP cierra la conexión de forma segura.
