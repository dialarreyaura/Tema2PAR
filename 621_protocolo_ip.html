6.2.1. Protocolo IP

El Protocolo de Internet (IP) es la piedra angular de las comunicaciones en redes, encargándose del direccionamiento y el enrutamiento de paquetes de datos entre dispositivos a través de redes interconectadas, incluidas LANs (redes locales) e internet. Funciona en la capa de red del modelo TCP/IP y es responsable de enviar los paquetes desde el origen hasta su destino, asegurando que los datos puedan viajar a través de múltiples redes.

El protocolo IP es sin conexión y no confiable por diseño, lo que significa que no garantiza la entrega de los paquetes ni asegura que lleguen en el mismo orden en que fueron enviados. Sin embargo, esta simplicidad y flexibilidad lo han convertido en el protocolo más utilizado en redes globales.

Formato del datagrama IP

El datagrama IP es la unidad básica de datos que viaja a través de redes IP. Se trata de un conjunto de datos que incluye una cabecera con información de control y una carga útil, que contiene los datos reales. El formato del datagrama IP varía ligeramente entre las versiones de IP (IPv4 e IPv6), pero a continuación se describe el formato de un datagrama IPv4 típico, que ha sido el estándar dominante durante muchos años:

Cabecera del datagrama IP (IPv4)
La cabecera IP es un bloque de datos de longitud fija (aunque puede extenderse con campos opcionales) que contiene toda la información necesaria para el enrutamiento y entrega del datagrama. La cabecera de un datagrama IPv4 tiene una longitud mínima de 20 bytes y está compuesta por varios campos:

  • Versión (4 bits): Indica la versión del protocolo IP. Para IPv4, este valor es 4.
    Longitud de la cabecera (4 bits): Indica el tamaño de la cabecera en palabras de 32 bits (4 bytes). El valor mínimo es 5, lo que equivale a 20 bytes.
  • Tipo de servicio (ToS) o DSCP (8 bits): Define la prioridad del paquete y el tipo de servicio que se espera. Puede utilizarse para dar preferencia a ciertos tipos de tráfico, como voz o video. De escasa utilidad en la actualidad. Su formato es:   

    • PRIO: Se utiliza en casos de congestión.

    • D: Dar prioridad al retardo.

    • T: Dar prioridad al throughput (tasa de transferencia).

    • R: Dar prioridad a la fiabilidad.

    • C: Dar prioridad al coste.

    • MBZ: Reservado para uso futuro. 1Bit.

Normalmente solo se pondrá a 1 uno de los campos D,T,R,C. El usuario decidirá a que quiere dar prioridad

  • Longitud total (16 bits): Especifica el tamaño total del datagrama, incluidas la cabecera y los datos. El tamaño máximo de un datagrama IP es de 65.535 bytes.
  • Identificación (16 bits): Un número único para identificar el datagrama. Se utiliza para fragmentar y reconstruir datagramas grandes.
  • Banderas (3 bits) y Desplazamiento de fragmentos (13 bits): Estos campos controlan la fragmentación del datagrama si es demasiado grande para una red específica. La fragmentación divide el datagrama en partes más pequeñas. Tiene los subcampos:
    • O: Reservado.
    • DF (Do not Fragment):Cuando el bit es 0, si permite fragmentar, cuando es 1 no permite fragmentar el datagrama.
    • MF (More Fragments): Cuando el bit es 1 quiere decir que se esperan mas fragmentos, y cuando el bit es 0, quiere decir que es el ultimo fragmento.
  • Desplazamiento de Fragmento (OFFSET): Número de fragmento dentro del datagrama. si pertenecen al mismo datagrama, el valor del campo IDENTIFICADOR sera el mismo.
  • Tiempo de vida (TTL, 8 bits): Limita la vida útil de un paquete en la red, evitando que un paquete se enrolle indefinidamente en bucles de enrutamiento. Cada vez que el paquete pasa por un router, su TTL se reduce en 1. Cuando llega a 0, el paquete se descarta.
  • Protocolo (8 bits): Identifica el protocolo de la capa de transporte que se utiliza (por ejemplo, TCP es el valor 6, UDP es el 17).
  • Checksum de la cabecera (16 bits): Un valor de verificación utilizado para detectar errores en la cabecera del datagrama.
  • Dirección IP de origen (32 bits): La dirección IP del dispositivo que envía el datagrama.
  • Dirección IP de destino (32 bits): La dirección IP del dispositivo al que se dirige el datagrama.
  • Opciones (longitud variable): Campo opcional que puede incluir información adicional, como registros de ruta o instrucciones de seguridad.

Carga útil del datagrama IP

Después de la cabecera, se encuentra la carga útil, que contiene los datos reales que deben ser transportados. En la mayoría de los casos, la carga útil es un segmento TCP o un datagrama UDP, aunque también puede ser otro tipo de información de red.

Las tramas físicas (de Ethernet) tienen un campo de datos que incluyen y transportan los datagramas IP. La longitud de este campo de datos no es indefinida debido a que está limitado por el diseño de la red.

El MTU ( Maximum Transmission Unit) de una red es la mayor cantidad de datos que puede transportar su trama física.

En la versión 4 del protocolo IP, IPv4, si el paquete a transmitir supera el tamaño máximo negociado (MTU) en el tramo de red por el que va a circular, podrá ser dividido en paquetes más pequeños, y reensamblado luego cuando sea necesario.

Estos fragmentos podrán ir cada uno por un camino diferente dependiendo de la congestión de las rutas encada momento. Si uno de los fragmentos se pierde, todo el paquete original se considerará perdido, y los restantes fragmentos se descartarán.

IP no establece un MTU máximo, pero sí establece un MTU mínimo de 576 bytes para su versión 4 y 1280 bytes (menor que la de Ethernet) para su versión 6.

Direccionamiento IP

El uso que se hace en la actualidad de las redes de datos impone la necesidad de un mecanismo de identificación unívoca de todas las máquinas de la red, independientemente de la tecnología del nivel de enlace de cada una.

Las direcciones IP son esenciales para el funcionamiento del protocolo IP, ya que identifican de manera única a cada dispositivo conectado a una red. Las direcciones IP pueden variar en formato y longitud según la versión del protocolo, pero todas cumplen con la misma función básica: identificar el origen y destino de los datagramas IP.

El nivel de red es el encargado de asignar direcciones a todas las máquinas de la red. Dicha dirección IP es el identificador de cada host dentro de su red.

Cada host conectado a una red tiene una dirección IP asignada, la cual debe ser distinta a todas las demás direcciones que estén vigentes en ese momento en el conjunto de redes visibles por el host.
Se utiliza el término host como equipo conectado a la red aunque se debería hablar de interfaz de red, que es lo que realmente conecta con la red. Cada interfaz físico conectado a Internet tiene asignada una dirección IP.

Un host puede tener más de una interfaz de red, normalmente para conectar a redes diferentes.
En el caso de Internet, no puede haber dos ordenadores con 2 direcciones IP (públicas) iguales. Pero sí podríamos tener dos ordenadores con la misma dirección IP siempre y cuando pertenezcan a redes independientes entre sí (sin ningún camino posible que las comunique).

Rango de direcciones

El rango de direcciones IP se refiere a los bloques de direcciones que pueden ser asignadas a dispositivos para que puedan comunicarse en una red utilizando el Protocolo de Internet (IP). Estos rangos están organizados de acuerdo con varias características, como la clase de la dirección y si las direcciones son públicas o privadas.

Las direcciones IP pueden clasificarse principalmente en dos tipos:

  1. Direcciones IP públicas, que son enrutables en internet.
  2. Direcciones IP privadas, que solo son válidas dentro de redes locales o privadas.

Direcciones IPv4
El espacio de direcciones IPv4 se basa en un rango de 32 bits, lo que significa que hay aproximadamente 4.294.967.296 (2^32) direcciones posibles. Estas direcciones se dividen en varios rangos, o bloques, algunos de los cuales son reservados para usos específicos, como las redes privadas, mientras que otros son asignados a dispositivos en internet a través de autoridades como IANA (Internet Assigned Numbers Authority).

Clases de direcciones IPv4

El sistema de direcciones IPv4 se divide en varias clases, que históricamente fueron usadas para asignar direcciones IP a diferentes tipos de redes según su tamaño. Las clases se definen por el valor de los bits más significativos de la dirección IP.

Clase A (0.0.0.0 – 127.255.255.255):

Primer octeto: 0-127
Usada para redes muy grandes, con hasta 16 millones de hosts posibles por red.
Direcciones de red: Los primeros 8 bits (primer octeto) corresponden a la parte de la red, dejando los 24 bits restantes para los hosts.
Ejemplo: 10.0.0.1
Rango privado: 10.0.0.0 – 10.255.255.255

Clase B (128.0.0.0 – 191.255.255.255):

Primer octeto: 128-191
Usada para redes medianas, con hasta 65.536 hosts por red.
Direcciones de red: Los primeros 16 bits corresponden a la parte de la red, mientras que los 16 bits restantes corresponden a los hosts.
Ejemplo: 172.16.0.1
Rango privado: 172.16.0.0 – 172.31.255.255

Clase C (192.0.0.0 – 223.255.255.255):

Primer octeto: 192-223
Usada para redes pequeñas, con hasta 256 hosts por red.
Direcciones de red: Los primeros 24 bits corresponden a la red, mientras que los 8 bits restantes son para los hosts.
Ejemplo: 192.168.0.1
Rango privado: 192.168.0.0 – 192.168.255.255

Clase D (224.0.0.0 – 239.255.255.255):

Usada para multicast, es decir, para la transmisión de datos a un grupo de hosts en lugar de un solo destinatario. No se utiliza para direccionar hosts individuales.

Clase E (240.0.0.0 – 255.255.255.255):

Reservada para fines experimentales y no se usa para el tráfico regular en internet.

Direcciones IPv4 privadas

Las direcciones privadas son aquellas que están reservadas para uso dentro de redes locales (LAN) y no son enrutables en internet. Estas direcciones se utilizan comúnmente en redes corporativas, hogares, y otras redes internas. La razón principal por la cual se utilizan direcciones privadas es la limitación del espacio de direcciones IPv4 públicas, lo que llevó a la implementación de tecnologías como NAT (Network Address Translation) para permitir que múltiples dispositivos dentro de una red local compartan una sola dirección IP pública al interactuar con internet.

Los rangos reservados para direcciones IPv4 privadas, según la RFC 1918, son:

Clase A privada:

Rango: 10.0.0.0 – 10.255.255.255
Tamaño: 16 millones de direcciones posibles.
Usos típicos: Grandes redes privadas, como las de grandes empresas.

Clase B privada:

Rango: 172.16.0.0 – 172.31.255.255
Tamaño: 1 millón de direcciones posibles.
Usos típicos: Redes privadas de tamaño medio.

Clase C privada:

Rango: 192.168.0.0 – 192.168.255.255
Tamaño: 65.536 direcciones posibles.
Usos típicos: Redes pequeñas, como redes domésticas o de pequeñas empresas.
Las direcciones IP privadas no pueden ser enrutadas directamente en internet. Para que los dispositivos con direcciones privadas accedan a internet, los routers utilizan NAT, que traduce las direcciones IP privadas en una dirección IP pública.

Direcciones IPv4 públicas
Las direcciones públicas son las que se utilizan para identificar dispositivos accesibles a través de internet. Estas direcciones deben ser únicas y son asignadas por una autoridad central, como IANA o los RIRs (Regional Internet Registries), que gestionan la asignación de bloques de direcciones IP en regiones específicas del mundo.

Direcciones públicas IPv4: Son aquellas que no están dentro de los rangos reservados para uso privado (RFC 1918) ni para propósitos especiales.
Por ejemplo: 8.8.8.8 (una dirección pública del DNS de Google).
Las direcciones públicas deben ser globalmente únicas para evitar conflictos en las redes mundiales.

Problema de agotamiento de direcciones IPv4
A medida que el uso de internet creció exponencialmente, el espacio de direcciones IPv4 se fue agotando rápidamente, lo que llevó a la introducción de soluciones temporales, como el uso generalizado de NAT, y finalmente al desarrollo de IPv6, una nueva versión del Protocolo de Internet con un espacio de direccionamiento mucho mayor.

Direcciones IPv6

IPv6 fue diseñado para reemplazar a IPv4 debido al problema de agotamiento de las direcciones. IPv6 tiene un espacio de 128 bits, lo que significa que soporta aproximadamente 340 sextillones de direcciones posibles, una cantidad mucho mayor que IPv4. Las direcciones IPv6 se representan en formato hexadecimal y están separadas por dos puntos.

Ejemplo de una dirección IPv6: 2001:0db8:85a3:0000:0000:8a2e:0370:7334

En IPv6, no se utiliza el concepto de clases como en IPv4, y en su lugar, se utilizan prefijos para definir la parte de la red y la parte del host. Sin embargo, al igual que en IPv4, IPv6 tiene rangos reservados para diferentes propósitos:

  • Direcciones unicast globales: Usadas para identificar dispositivos en internet, de manera similar a las direcciones IPv4 públicas.
  • Direcciones link-local: Usadas para comunicación dentro de una red local, como la configuración automática de dispositivos en la misma red.
  • Direcciones multicast: Usadas para enviar paquetes a múltiples destinos al mismo tiempo, lo que es útil en redes de gran escala.
  • Direcciones anycast: Permiten que un paquete sea entregado al dispositivo más cercano de un grupo de destinatarios, algo útil en servicios distribuidos como los DNS.

Rangos especiales en IPv4 y IPv6
Además de las direcciones privadas y públicas, tanto IPv4 como IPv6 tienen ciertos rangos reservados para usos especiales:

  • Dirección de loopback (IPv4: 127.0.0.1, IPv6: ::1): Usada para que un dispositivo se comunique consigo mismo. Esto es útil en pruebas de red y diagnósticos.
  • Direcciones de enlace-local (IPv4: 169.254.0.0/16, IPv6: fe80::/10): Utilizadas para comunicaciones locales dentro de una red, cuando no hay un servidor DHCP disponible para asignar una dirección IP.

A su vez, las direcciones IP pueden ser:

  • Direcciones IP estáticas (fijas): Un host que se conecte a la red con dirección IP estática siempre lo hará con una misma IP. Las direcciones IP públicas estáticas son las que utilizan los servidores de Internet con objeto de que estén siempre localizables por los usuarios de Internet. Estas direcciones hay que contratarlas.
  • Direcciones IP dinámicas: Un host que se conecte a la red mediante dirección IP dinámica, cada vez lo hará con una dirección IP distinta. Las direcciones IP públicas dinámicas son las que se utilizan en las conexiones a Internet mediante un módem/cable-módem/router. Los proveedores de Internet utilizan direcciones IP dinámicas debido a que tienen más clientes que direcciones IP (es muy improbable que todos se conecten a la vez).
