6.2.1.2. Tabla de encaminamiento

Imaginemos que tenemos una red de ordenadores interconectados entre sí. Cuando un ordenador quiere enviar datos a otro, esos datos deben viajar a través de una serie de dispositivos de red, como routers, que se encargan de guiar la información hasta su destino. Este proceso de guiar los datos por la red se llama encaminamiento.

El encaminamiento y las tablas de encaminamiento son conceptos fundamentales en redes informáticas, ya que son los mecanismos que permiten que los datos (paquetes) viajen de un dispositivo a otro a través de múltiples redes interconectadas, como ocurre en Internet.

¿Qué es el encaminamiento?

El encaminamiento es el proceso por el cual los dispositivos de red (principalmente routers o enrutadores) determinan la mejor ruta o camino por el que debe viajar un paquete de datos para llegar desde su origen hasta su destino, a través de una red o una serie de redes. En otras palabras, cuando envías información desde tu computadora a un servidor, los routers en el camino deben decidir cómo hacer que el paquete llegue de manera eficiente al destino correcto. Cada vez que un paquete llega a un router, este debe decidir a cuál de sus salidas (interfaces) enviarlo, basándose en la dirección IP de destino del paquete. Esto es esencial para garantizar que los datos viajen eficientemente desde el origen hasta el destino, incluso si están en redes diferentes o muy distantes.

¿Cómo funciona el encaminamiento?

  1. Los routers son dispositivos encargados de tomar decisiones sobre el camino que debe seguir un paquete.
  2. Cada router tiene una tabla de encaminamiento, que es como un "mapa" donde el router guarda la información sobre las posibles rutas o caminos hacia diferentes redes.
  3. Cuando un router recibe un paquete de datos, mira en su tabla de encaminamiento para ver cuál es la mejor ruta que debe seguir ese paquete para llegar a su destino.
  4. Los paquetes se reenvían de un router a otro, hasta que alcanzan la red destino.

Elementos necesarios para el enrutamiento.

  1. Direcciones IP: Cada dispositivo en una red tiene una dirección IP única. Cuando un dispositivo envía datos a otro, el paquete contiene la dirección IP del destinatario.
  2. Decisiones de encaminamiento: Los routers en la red revisan la dirección IP de destino y deciden por cuál de sus "caminos" deben enviar el paquete para acercarlo a su destino final.
  3. Algoritmos de encaminamiento: Los routers utilizan algoritmos para decidir la mejor ruta posible. Algunos factores que pueden influir en esta decisión incluyen:
    • La distancia a la red de destino.
    • La cantidad de tráfico en una ruta.
    • La confiabilidad de los enlaces.

Tipos de encaminamiento

  • Encaminamiento directo: Ocurre cuando el dispositivo origen y el dispositivo destino están en la misma red local. En este caso, no es necesario pasar por un router. El dispositivo puede enviar el paquete directamente al destinatario.
  • Encaminamiento indirecto: Ocurre cuando el dispositivo origen y el destino están en redes diferentes. Aquí, el paquete necesita pasar por uno o más routers intermedios para llegar a la red destino.

Tablas de encaminamiento

Una tabla de encaminamiento es como un mapa o un directorio que el router utiliza para tomar decisiones. La tabla de encaminamiento es una base de datos que utiliza un router para decidir por dónde enviar un paquete de datos. En términos simples, es una lista que indica qué redes están disponibles y cómo llegar a ellas. Cada fila en la tabla de encaminamiento se conoce como una entrada o ruta, y contiene información clave que ayuda a tomar las decisiones de encaminamiento. En esta tabla, el router guarda información sobre:

  • Redes conocidas: Qué redes conoce y cómo llegar a ellas.
  • Rutas: Por qué camino (o interfaz) debe enviar los datos para llegar a una red específica.
  • Distancia: Cuál es la distancia o "costo" para llegar a esa red. A veces, esto se refiere al número de "saltos" entre routers que se requieren para llegar al destino.

Siendo más específicos, cada entrada en una tabla de enrutamiento tiene al menos los siguientes campos:

  • Destino de la red: Indica el rango de direcciones IP que corresponden a una red específica a la que el router puede enviar datos. Por ejemplo, la red 192.168.1.0/24 incluye todas las direcciones IP desde 192.168.1.1 hasta 192.168.1.254.
  • Máscara de subred: Define qué parte de la dirección IP se refiere a la red y qué parte se refiere a los hosts dentro de esa red. En el ejemplo anterior, la máscara de subred 255.255.255.0 indica que los primeros 24 bits (192.168.1) son la parte de red y el último byte es para los hosts.
  • Puerta de enlace (gateway): Es la dirección IP del siguiente dispositivo (normalmente otro router) al que el paquete debe ser enviado para continuar su camino hacia la red de destino. Si el destino está en la red local, no se necesita un gateway.
  • Interfaz de salida: Indica por qué puerto físico o interfaz de red (como Ethernet o Wi-Fi) debe enviarse el paquete para llegar al siguiente salto en su camino.
  • Métrica: Un valor que indica el costo de usar una determinada ruta. Puede basarse en diferentes factores, como la cantidad de routers intermedios, el ancho de banda o el tiempo de respuesta. Las rutas con métricas más bajas son preferidas porque se consideran más eficientes.

Ejemplo práctico de tabla de encaminamiento

Imagina que tienes una red doméstica con un router que conecta varios dispositivos a Internet. A continuación, se muestra un ejemplo simplificado de una tabla de encaminamiento en el router de tu red:

Destino Máscara Puerta de enlace Interfaz de Salida Métrica
192.168.1.0 255.255.255.0 ---- eth0 1
10.0.0.0 255.0.0.0 192.168.1.1 eth0 2
0.0.0.0 0.0.0.0 1+92.168.1.1 eth0 10

Explicación de cada entrada:

  • 192.168.1.0 / 255.255.255.0: Esta es la red local. No necesita una puerta de enlace (gateway) porque el router puede comunicarse directamente con los dispositivos en esta red. La interfaz de salida es eth0, que es el puerto Ethernet del router, y tiene una métrica de 1 (es la ruta más rápida y directa para paquetes destinados a esa red).
  • 10.0.0.0 / 255.0.0.0: Si se quiere enviar un paquete a una dirección IP en la red 10.0.0.0 (otra red), el paquete debe ser enviado a través de la puerta de enlace 192.168.1.1 (un router intermedio) usando la interfaz eth0. La métrica es 2, lo que significa que esta ruta es más "costosa" en comparación con la red local, pero sigue siendo viable.
  • 0.0.0.0 / 0.0.0.0: Esta es una ruta por defecto. Si el router no sabe cómo llegar a una red específica, enviará el paquete a la puerta de enlace 192.168.1.1 (generalmente la conexión con el proveedor de servicios de Internet). La métrica de 10 indica que esta es una opción menos preferida comparada con las redes locales o cercanas.

¿Cómo decide un router qué ruta usar?
Cuando un router recibe un paquete de datos, sigue estos pasos:

  1. Examina la dirección IP de destino del paquete.
  2. Busca en la tabla de encaminamiento para ver si hay una ruta específica que coincida con esa dirección IP.
  3. Si encuentra una coincidencia exacta, sigue esa ruta. Si no hay coincidencia exacta, verifica si hay una ruta por defecto (normalmente indicada como 0.0.0.0).
  4. Si encuentra la mejor ruta, envía el paquete a través de la interfaz especificada hacia la puerta de enlace correcta.

Ejemplo de cómo funciona el encaminamiento:
Supongamos que un dispositivo con la dirección IP 192.168.1.100 en tu red doméstica quiere acceder a una página web en un servidor con la dirección 10.0.0.200.

  1. El dispositivo envía el paquete a su router, ya que el destino está fuera de su red local.
  2. El router examina su tabla de encaminamiento y encuentra la entrada que corresponde a la red 10.0.0.0/8.
  3. El router envía el paquete al siguiente router en la ruta (puerta de enlace 192.168.1.1).
  4. El paquete continúa a través de routers intermedios hasta que llega al servidor con dirección 10.0.0.200.
