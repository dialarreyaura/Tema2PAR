5.3. La capa de red

La Capa de Red es la tercera capa del modelo OSI y es responsable de determinar cómo se enrutan los datos desde el origen hasta el destino a través de múltiples redes interconectadas, lo que comúnmente se conoce como enrutamiento. Esta capa se encarga de gestionar la dirección lógica, la encaminación y la transmisión de paquetes de datos entre diferentes redes.

El principal propósito de la Capa de Red es garantizar que los datos enviados desde un dispositivo en una red lleguen al dispositivo adecuado en otra red, a través de múltiples saltos intermedios.

Principales Funciones de la Capa de Red

1. Encaminamiento (Routing):

Selecciona la mejor ruta o camino que un paquete de datos debe tomar para llegar a su destino a través de una o más redes interconectadas. Utiliza protocolos de enrutamiento como RIP, OSPF, BGP, EIGRP, entre otros, para determinar estas rutas de manera dinámica.

2. Direccionamiento Lógico:

La Capa de Red utiliza direcciones lógicas, como direcciones IP (IPv4 e IPv6), para identificar de manera única cada dispositivo en una red. Estas direcciones permiten que los dispositivos sean localizados más allá de sus redes locales.

3. Fragmentación y Reensamblaje:

Cuando un paquete es demasiado grande para ser transmitido por el medio de transmisión de la red de destino, la Capa de Red puede fragmentar el paquete en partes más pequeñas y luego reensamblarlo en el destino.

4. Control de Congestión:

Puede ayudar a gestionar y reducir la congestión en la red utilizando técnicas como el control de tráfico, colas de prioridad, y políticas de calidad de servicio (QoS).

5. Manejo de Errores y Diagnóstico:

Aunque no se encarga directamente de la detección y corrección de errores (esto es principalmente responsabilidad de la Capa de Enlace de Datos), la Capa de Red proporciona algunos mecanismos de diagnóstico y notificación de errores a través de protocolos como ICMP (Internet Control Message Protocol).

6. Conmutación de Paquetes (Packet Switching):

La Capa de Red utiliza conmutación de paquetes, lo que significa que los datos se envían en bloques discretos llamados paquetes. Cada paquete es independiente y puede tomar diferentes rutas para llegar al destino.

Protocolos Clave en la Capa de Red

1. IP (Internet Protocol):

  • IPv4: Protocolo de red más utilizado, que utiliza direcciones de 32 bits (formato decimal) y permite aproximadamente 4,300 millones de direcciones únicas. Implementa funciones básicas como el direccionamiento, la fragmentación, y el reensamblaje.
  • IPv6: Introducido para superar las limitaciones de IPv4, utiliza direcciones de 128 bits (formato hexadecimal), permitiendo un número prácticamente ilimitado de direcciones únicas. También ofrece mejoras en eficiencia de enrutamiento, seguridad integrada, y gestión de tráfico.

2. ICMP (Internet Control Message Protocol):

Protocolo utilizado para enviar mensajes de control y error entre dispositivos de red. Es fundamental para diagnósticos de red (por ejemplo, comandos como ping y traceroute) y para informar sobre problemas como destinos inaccesibles o tiempos de espera agotados.

3. ARP (Address Resolution Protocol):

Utilizado para mapear direcciones IP a direcciones MAC (de la Capa 2) dentro de una red local. Es esencial para la comunicación dentro de redes locales donde se necesita conocer la dirección física del dispositivo.

4. RARP (Reverse Address Resolution Protocol):

Utilizado para mapear una dirección MAC a una dirección IP. Es menos utilizado hoy en día debido a otros protocolos más eficientes como DHCP.

5. Protocolo de Encapsulación:

Proporciona el encapsulamiento de datos de capas superiores en paquetes IP para su transmisión a través de la red.

Protocolos de Enrutamiento:

  • RIP (Routing Information Protocol): Protocolo de enrutamiento basado en el vector de distancia que utiliza el número de saltos como métrica para determinar la ruta más corta.
  • OSPF (Open Shortest Path First): Protocolo de enrutamiento basado en el estado de enlace que utiliza el algoritmo de Dijkstra para calcular la ruta más corta.
  • BGP (Border Gateway Protocol): Protocolo de enrutamiento de pasarela exterior utilizado para enrutamiento entre sistemas autónomos (AS) en Internet. Es el protocolo que mantiene la tabla de enrutamiento global de Internet.

Proceso de Encapsulación y Desencapsulación

En la Capa de Red, los datos provenientes de la Capa de Transporte son encapsulados en paquetes IP. Cada paquete tiene un encabezado IP que contiene información vital como la dirección IP de origen y destino, el TTL (Time To Live), y los campos de control de calidad.
Cuando un paquete llega a su destino, la Capa de Red en el dispositivo receptor desencapsula los datos y los pasa a la Capa de Transporte.

Encabezado IP (Formato del Paquete IP)

El encabezado IP es fundamental para la operación de la Capa de Red. Contiene múltiples campos importantes, entre ellos:

Versión: Indica si es IPv4 o IPv6.
Longitud del Encabezado: Tamaño del encabezado IP.
Tipo de Servicio (ToS) / Clase de Tráfico: Prioridad del paquete para QoS.
Longitud Total: Tamaño total del paquete (encabezado + datos).
Identificación, Banderas, Fragmento Desplazado: Utilizados para fragmentación y reensamblaje de paquetes.
TTL (Time To Live): Evita bucles infinitos en la red, decreciendo en cada salto.
Protocolo: Protocolo de la capa superior (por ejemplo, TCP o UDP).
Checksum del Encabezado: Verificación de integridad del encabezado.
Dirección IP de Origen y Destino: Direcciones IP de los dispositivos de red.

Ejemplos Prácticos de la Capa de Red

  • Internet: Uso del protocolo IP para enrutar paquetes a través de diferentes redes, desde la computadora del usuario final hasta servidores web en todo el mundo.
  • Enrutadores (Routers): Dispositivos de red que operan principalmente en la Capa de Red, utilizando protocolos de enrutamiento para decidir el mejor camino para los datos.

Importancia de la Capa de Red

La importancia de la capa de red radica en los siguientes aspectos:

  • Interconexión de Redes: Permite que diferentes redes se interconecten y se comuniquen entre sí.
  • Enrutamiento Eficiente: Selecciona la mejor ruta para el tráfico de red, optimizando el uso de los recursos de red.
  • Escalabilidad y Flexibilidad: La Capa de Red es escalable y permite la expansión de la red sin problemas significativos de compatibilidad.
  • Base para Internet: Es fundamental para la funcionalidad de Internet y las redes de área amplia (WAN).

La Capa de Red del modelo OSI es fundamental para la conectividad global de redes, permitiendo la comunicación y transmisión de datos a través de múltiples redes. Su enfoque en el enrutamiento, direccionamiento lógico, y control de tráfico hace posible la operación efectiva y eficiente de redes grandes y complejas como Internet.
