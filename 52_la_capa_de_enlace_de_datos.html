5.2. La capa de enlace de Datos

La Capa de Enlace de Datos es la segunda capa del Modelo OSI. Su función principal es establecer, mantener y terminar la conexión de enlace de datos entre dos dispositivos físicamente conectados. Esta capa se encarga de la transferencia fiable de datos a través del medio físico, manejando la detección y corrección de errores, el control de flujo, y la dirección física. Su objetivo es asegurar que los datos enviados desde la capa de red de un dispositivo lleguen de manera confiable a la capa de red del dispositivo de destino.

  • Objetivos de la Capa de Enlace de Datos:
    Proporcionar un medio de comunicación fiable entre dos nodos conectados directamente.
  • Gestionar el control de acceso al medio para evitar colisiones y pérdidas de datos.
  • Garantizar la integridad de los datos mediante técnicas de detección y corrección de errores.
  • Asegurar el control de flujo para prevenir la sobrecarga del receptor.

Subcapas de la Capa de Enlace de Datos

La Capa de Enlace de Datos se divide en dos subcapas para manejar sus diversas responsabilidades:

Subcapa de Control de Enlace Lógico (LLC - Logical Link Control): Proporciona la interfaz con la Capa de Red superior. Se encarga de la gestión del enlace lógico, la multiplexación de protocolos de red y el control de flujo. Implementa mecanismos para establecer, mantener y finalizar los enlaces lógicos.

Subcapa de Control de Acceso al Medio (MAC - Media Access Control): Se ocupa del acceso al medio de transmisión, controlando cómo los dispositivos en una red comparten el medio físico. Proporciona direcciones físicas (direcciones MAC) y se encarga de la detección de colisiones y resolución de conflictos. Define los formatos de tramas y cómo se estructuran los datos para la transmisión.

Responsabilidades y Funciones Clave

 Las responsabilidades y funciones clave en esta capa son las siguientes:
1. Enmarcado, tramado (Framing):

  • La Capa de Enlace de Datos organiza los bits recibidos de la Capa Física en tramas (frames), que son estructuras de datos con una cabecera, datos y un tráiler.
  • La cabecera de la trama contiene la dirección MAC de origen y destino, mientras que el tráiler incluye un código de detección de errores (como CRC).

2. Detección y Corrección de Errores:

Utiliza métodos como Checksum, CRC (Cyclic Redundancy Check) y Hamming Code para la detección de errores en la transmisión de datos.
Algunas técnicas de corrección, como ARQ (Automatic Repeat reQuest), permiten que el receptor solicite la retransmisión de datos cuando se detectan errores.

3. Control de Flujo:

  • Asegura que el emisor no envíe datos más rápido de lo que el receptor puede procesarlos.
  • Protocolos como Stop-and-Wait, Sliding Window y Go-Back-N se utilizan para gestionar el control de flujo.

4. Control de Acceso al Medio (MAC):

  • La Capa de Enlace define cómo los dispositivos acceden al medio de red y previene las colisiones en un entorno de red compartida.
  • Protocolos de acceso como CSMA/CD (Carrier Sense Multiple Access with Collision Detection), utilizado en Ethernet, o CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance), utilizado en Wi-Fi, gestionan el acceso al medio.

5. Direcciones MAC (Media Access Control):

  • La capa de enlace utiliza direcciones MAC, que son direcciones únicas de 48 bits asignadas a cada interfaz de red, para identificar los dispositivos en la red local.
  • Facilita la entrega de datos entre dispositivos en la misma red o segmento.

6. Topologías Lógicas de Red:

Define la topología lógica de la red (cómo los datos fluyen entre dispositivos), que puede ser diferente de la topología física. Ejemplos de topologías lógicas incluyen bus, anillo, y estrella.

7. Conmutación de Tramas:

Utiliza switches para segmentar redes y proporcionar conmutación de tramas en la Capa 2, lo que permite la creación de dominios de colisión separados y mejora la eficiencia.

8. Control de Enlace de Datos:

Gestiona los enlaces punto a punto y punto a multipunto entre dispositivos, asegurando la transmisión confiable de datos.

Protocolos y Tecnologías en la Capa de Enlace de Datos

Ethernet (IEEE 802.3): Uno de los estándares más comunes para la Capa de Enlace de Datos, define tanto la subcapa MAC como algunas especificaciones de la capa física. Utiliza CSMA/CD para el control de acceso al medio.
Wi-Fi (IEEE 802.11): Utiliza CSMA/CA para evitar colisiones en redes inalámbricas y define cómo los dispositivos acceden al medio compartido.
PPP (Point-to-Point Protocol): Protocolo de enlace de datos utilizado para conexiones punto a punto, como en conexiones de Internet a través de módems.
HDLC (High-Level Data Link Control): Protocolo de control de enlace de datos para conexiones WAN que proporciona encapsulamiento de datos y control de errores.
Frame Relay: Protocolo WAN utilizado para la transmisión de tramas en redes de área extensa.

Ejemplos Prácticos

  • En una red Ethernet, la Capa de Enlace de Datos utiliza direcciones MAC para enviar una trama desde una computadora a otra dentro del mismo dominio de transmisión.
  • En una red Wi-Fi, cuando un dispositivo desea transmitir datos, primero escucha el canal para ver si está libre (CSMA/CA) y, si lo está, envía una trama de solicitud antes de transmitir los datos.

Importancia de la Capa de Enlace de Datos

La importancia de la capa de enlace de datos radique en que:

  • Es fundamental para la Comunicación Local: La Capa de Enlace es crucial para el funcionamiento de redes locales (LAN) y es la base sobre la que se construyen capas superiores.
  • Permite la Optimización de Red y Seguridad: Gestionar adecuadamente la Capa de Enlace puede mejorar la eficiencia de la red, reducir colisiones y proporcionar una base para la seguridad de la red mediante técnicas como listas de control de acceso (ACLs) a nivel de conmutadores.
  • Y es la Base para Redes Confiables: Proporciona mecanismos para la entrega confiable de datos, lo que es esencial para las aplicaciones que dependen de una comunicación continua y precisa.
