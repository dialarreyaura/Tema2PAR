6.4. La capa de aplicación

La capa de aplicación es la capa más alta en el modelo TCP/IP, donde ocurren las interacciones directas entre las aplicaciones del usuario y los servicios de red. Su principal función es ofrecer las interfaces y los protocolos que permiten que las aplicaciones utilicen la red para enviar y recibir datos. Aquí es donde se sitúan los protocolos que usamos todos los días, como HTTP, FTP, SMTP, DNS, entre otros.

La capa de aplicación facilita la comunicación entre procesos a través de la red, traduciendo las acciones del usuario en solicitudes a otros dispositivos o servidores. Los protocolos de esta capa gestionan la forma en la que las aplicaciones acceden a los servicios de red y cómo los datos se presentan al usuario. También se asegura de que los datos sean interpretados correctamente por la aplicación receptora.

Funciones principales

  • Comunicación entre aplicaciones: La capa de aplicación permite que dos aplicaciones en diferentes dispositivos se comuniquen entre sí. Por ejemplo, un navegador web en tu computadora (cliente) se comunica con un servidor web remoto para descargar una página web.
  • Interacción con la capa de transporte: Aunque la capa de aplicación no gestiona directamente los aspectos técnicos de la transmisión de datos (esto lo hace la capa de transporte con protocolos como TCP o UDP), sí utiliza los servicios proporcionados por ella. Los datos generados en la capa de aplicación se encapsulan en paquetes por la capa de transporte.
  • Control de la presentación de datos: Algunos protocolos en esta capa también se ocupan de cuestiones como la codificación y la presentación de los datos. Por ejemplo, en una transferencia FTP (File Transfer Protocol), se decide si los archivos se transfieren en formato binario o ASCII.
  • Proporciona protocolos específicos para servicios: Cada aplicación o servicio utiliza un protocolo específico para interactuar con la red. Estos protocolos están diseñados para cumplir con los requisitos de diferentes tipos de comunicación, como la transferencia de archivos, el envío de correos electrónicos o la navegación web.

Protocolos clave en la capa de aplicación

En esta capa hay una serie de protocolos muy conocidos, entre los protocolos más comunes que operan en la capa de aplicación:

  • HTTP (Hypertext Transfer Protocol): Utilizado por navegadores web para recuperar información de servidores web. Es el protocolo principal para la navegación web y opera en el puerto 80 (HTTP) o 443 (HTTPS) para conexiones seguras.
  • FTP (File Transfer Protocol): Se utiliza para la transferencia de archivos entre un cliente y un servidor. Opera en el puerto 21 para establecer la conexión de control y en un rango de puertos para la transferencia de datos.
  • SMTP (Simple Mail Transfer Protocol): Se utiliza para el envío de correos electrónicos desde un cliente de correo a un servidor de correo, o entre servidores de correo. Funciona principalmente en el puerto 25.
  • DNS (Domain Name System): Traducir los nombres de dominio (por ejemplo, www.google.com) en direcciones IP que pueden ser utilizadas por los dispositivos de la red. Opera en el puerto 53, utilizando tanto TCP como UDP.
  • Telnet: Permite a los usuarios controlar remotamente un dispositivo en la red a través de una sesión interactiva de texto. Opera en el puerto 23, aunque ha sido reemplazado en muchos entornos por SSH debido a preocupaciones de seguridad.
  • SSH (Secure Shell): Similar a Telnet, pero añade una capa de cifrado para garantizar la seguridad durante las conexiones remotas. Funciona en el puerto 22.
  • POP3/IMAP (Post Office Protocol/Internet Message Access Protocol): Protocolos utilizados para la recuperación de correos electrónicos desde un servidor de correo hacia el cliente de correo. POP3 usa el puerto 110 y IMAP el puerto 143, aunque las versiones seguras POP3S e IMAPS utilizan los puertos 995 y 993 respectivamente.

Ejemplo práctico

Cuando usas tu navegador para acceder a una página web (por ejemplo, www.example.com), ocurre lo siguiente en la capa de aplicación:

  1. El navegador (una aplicación de usuario) realiza una solicitud HTTP para obtener el contenido de la página web. Esto implica enviar una solicitud GET al servidor que aloja el sitio.
  2. El navegador usa el DNS para traducir el nombre de dominio “www.example.com” a una dirección IP (por ejemplo, 192.168.1.10).
  3. Una vez obtenida la IP del servidor, la solicitud HTTP es enviada a la dirección IP del servidor a través del puerto 80 (HTTP) o 443 (HTTPS), usando TCP como protocolo de transporte.
  4. El servidor web responde enviando de vuelta los datos correspondientes a la página solicitada, y el navegador los presenta al usuario.

Debes recordar

La capa de aplicación es crucial porque es donde interactúan las aplicaciones que utilizamos en nuestra vida cotidiana. Es la parte del modelo TCP/IP más visible para los usuarios finales, ya que proporciona las interfaces y protocolos que facilitan el uso de la red. Sin los protocolos de la capa de aplicación, los usuarios no podrían acceder a servicios como el correo electrónico, la web o la transferencia de archivos.

Resumen

  • La capa de aplicación se encarga de proporcionar los protocolos que permiten a las aplicaciones comunicarse a través de la red.
  • Facilita la interacción entre los procesos de red y las aplicaciones de los usuarios.
  • Algunos de los protocolos más conocidos incluyen HTTP, FTP, SMTP, DNS, SSH, entre otros.
  • A través de esta capa, los datos son preparados y enviados para ser transportados por las capas inferiores del modelo TCP/IP.

Este nivel es donde la red cobra "vida" desde el punto de vista del usuario final, ya que maneja los servicios y aplicaciones que usamos todos los días.
